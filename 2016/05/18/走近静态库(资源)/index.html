走近静态库(资源)

2016-05-18 Objective-C

0x01 bundle

bundle实质是其实就是一个有后缀(.bundle)的文件夹

在制作framework或者.a时,由于一些资源文件(图片,音频,storyboard,xib,xcassets,plist…等等)并不能直接编译到可执行文件中,这时就需要我们提供一个容器来管理这些资源文件,这就是我们今天要说的bundle

0x02 How to make it

2.1 创建bundle工程

由于苹果爹并没有提供iOS平台制作bundle的工程模板,只提供了OS X系统的工程模板,所以第一步,我们先创建一个OS X系统下的bundle工程模板.

创建好工程后我们需要做以下设置,使之能完美运行在iOS平台:

  • 默认情况下，有两种resolutions的图片可以产生一些有趣的现象。例如，当你导入一个retina @2x版本的图片时，普通版的和Retina版的将会合并成一个多resolution的TIFF（标签图像文件格式，Tagged Image File Format）。这不是一件好事。搜索hidpi将COMBINE_HIDPI_IMAGES设置为NO。

  • 确保Build Setting -> targeted device family设置为1,2, 或者将工程通用设置里的 Deployment info里的devices设置为Universal,这样使你的bundle在iPad和iPhone下都可以使用,假如只设置了iPhone也就是 targeted device family为1的时候,那么当iPad工程使用这个bundle的时候,上传AppStore会被检测出错误,导致上不了架

2.2 添加资源文件

然后将要放入bundle的资源文件添加到工程中,并确认这些资源文件都加入到工程的 Build Phases -> Copy Rundle Resources中

2.3 编译打包

编译过程中我们需要了解一下下面几点

  • .xcassets将会被编译成为 .car文件

  • .storyboard文件将会被编译成 .storyboardc文件,storyboard文件中设置图片可以直接引用.xcassets中的图片

  • .xib文件将会被编译成 .nib文件

  • .storyboard文件和.xib文件可以直接与库中的文件连线操作,这些信息将会编译进相应的.storyboardc和.nib

  • 不要将bundle放入到framework文件夹下一起提供,使用者将framework引入工程后,framework里的bundle并不会拷贝到最终的程序包里,导致引用不到资源
